Research Story
After Teduh successfully launched in the AppStore, we received a lot of feedback from our users, one of those feedbacks is: Using SwiftUI makes users with iOS under iOS 13 unable to use our application. We also see several weaknesses in our application, one of our weaknesses is on the user experience side. We see that even though the user experience of our application is quite intuitive, the user cannot interact more when running a session in Teduh.
Therefore, in the final challenge at the Apple Developer Academy, we decided to continue the development of Teduh. The first step we take is to make as many guiding questions as possible regarding the advantages and disadvantages of shade. After that, we tried to compile the guiding question into interview questions to ask our users.
We interviewed six of our users. We try to ask simple but fundamental questions like: "Would you like to use a recording session without having to go through a grounding session?" From the interview, we got data that: Our users want freedom when using Teduh.

Technical Story

Finally, we totally changed Teduh's user interface and user experience. One of the big changes we made was: We allowed our users to write journals in the Teduh, without even having to follow a session, added a Siri Shortcut, added dark mode, and added language settings using localization. Due to the weakness of SwiftUI that it can't be run on iOS 13 and below and many of our users who want Teduh to be present in those iOS versions, we are redeveloping this application using the UI Kit. Here are my responsibilities as an iOS Engineer at Teduh:

  • Data Storage System using CoreData
    • I was again responsible for creating a data storage system for journal results, audio, journal dates, but in this project, I used core data to store more data effectively. I divide the data entities in the shade application into these several attributes:
  • audioData: String -> Save Audio Title for fetching with URL.
  • audioEmpty: Boolean -> Indicates whether the journal has audio
  • notesData: String -> a string which is the user's writing in textview
  • notesEmpty: Boolean -> Indicates whether the journal has text
  • sessionDate: Date -> Save Data which is the time when the user performs a session record
  • sessionName: String -> Saves the session name which the user chooses when saving the journal

  • UIComponent On RecordJournal and JournalViewController

    • I am also responsible for creating all views in the journal session page and the journal view. I use segmented control, textview, and scrollview on the journal session page. On the journal view page,I am asked to create a tableview so that it can be expanded when selected. I tried to use the lines of text in the tableview to change the size of the dynamic tableview. This allows the tableview to be expanded when touched.

  • Siri Shortcut

    • Lastly, I am responsible for creating a series of shortcuts on Teduh. The Siri shortcut will automatically open a user-selected first aid session in the Teduh. The existence of a series of shortcuts allows users to quickly start a session if they experience a panic attack. I learned to use UIIntents in creating a series of shortcuts. I also learned about the app cycle when installing a series shortcut in the delegate and app delegate scenes.
